本项目所有 AI 开发工作必须遵守以下规范。

### 1.1 Think Before Coding
- 不假设、不隐藏困惑、明确说出权衡点
- 存在多种理解时，列出选项而不是默默选一个
- 如果有更简单的方法，直接说

### 1.2 Simplicity First
- 最少代码解决问题，不做推测性设计
- 不为单用代码造抽象
- 不添加未被要求的"灵活性"或"可配置性"
- 如果 200 行能写成 50 行，重写它

### 1.3 Surgical Changes
- 只碰必须碰的代码
- 不改无关的代码、注释、格式
- 匹配现有风格
- 自己产生的废弃引用自己清理，不删已有的死代码

### 1.4 Goal-Driven Execution
- 把任务转化为可验证的目标
- 多步骤任务给出简要计划：
  ```
  1. [Step] → verify: [check]
  2. [Step] → verify: [check]
  ```
