## 药物发现与药物化学

### 案例 1：肺癌新型 EGFR 抑制剂的发现

**目标**：识别与现有药物相比具有改进性质的新型 EGFR 小分子抑制剂。

**使用的技能**：
- `database-lookup` — 查询 ChEMBL、PubChem、COSMIC、AlphaFold DB
- `paper-lookup` — PubMed 文献检索
- `rdkit` — 分子性质分析
- `datamol` — 生成类似物
- `medchem` — 药物化学过滤器
- `molfeat` — 分子特征化
- `diffdock` — 分子对接
- `deepchem` — 性质预测
- `torchdrug` — 分子图神经网络
- `scientific-visualization` — 创建图表
- `clinical-reports` — 生成 PDF 报告

**工作流程**：

```
步骤 1：查询 ChEMBL 获取高效力 EGFR 抑制剂
- 搜索靶向 EGFR 的化合物（CHEMBL203）
- 过滤：IC50 < 50 nM，pChEMBL 值 > 7
- 提取 SMILES 字符串和活性数据
- 导出到 DataFrame 进行分析

步骤 2：分析构效关系
- 用 RDKit 加载化合物
- 计算分子描述符（MW、LogP、TPSA、HBD、HBA）
- 生成分子指纹（半径=2，2048 位）
- 进行层次聚类识别骨架
- 可视化带活性注释的顶部骨架

步骤 3：从 COSMIC 识别耐药突变
- 查询 COSMIC 获取肺癌中 EGFR 突变
- 关注看门人突变（T790M、C797S）
- 提取突变频率和临床意义
- 与 PubMed 文献交叉参考

步骤 4：从 AlphaFold 获取 EGFR 结构
- 下载 EGFR 激酶域的 AlphaFold 预测
- 或从 PDB 获取实验结构
- 准备对接结构（添加氢原子、优化）

步骤 5：使用 datamol 生成新型类似物
- 从 ChEMBL 分析中选择前 5 个骨架
- 使用骨架修饰每个骨架生成 100 个类似物
- 应用 Lipinski 五规则过滤
- 确保合成可及性（SA 分数 < 4）
- 检查 PAINS 和不想要的子结构

步骤 6：使用 DeepChem 预测性质
- 在 ChEMBL EGFR 数据上训练图卷积模型
- 预测生成类似物的 pIC50
- 预测 ADMET 性质（溶解度、渗透性、hERG）
- 按预测效力和类药性排序候选物

步骤 7：使用 DiffDock 进行虚拟筛选
- 对前 50 个候选物进行分子对接
- 对接到野生型 EGFR 和 T790M 突变体
- 计算结合能和相互作用模式
- 识别对两种形式都有利结合的化合物

步骤 8：搜索 PubChem 获取商业可用性
- 按 InChI 键查询前 10 个候选物
- 检查供应商信息和购买选项
- 识别近似类似物（如果精确匹配不可用）

步骤 9：使用 PubMed 进行文献验证
- 搜索顶部骨架的先前技术
- 总结相关发现和潜在缺陷

步骤 10：创建综合报告
- 生成热门命中的 2D 结构可视化
- 创建散点图：MW vs LogP、TPSA vs 效力
- 生成前 3 个化合物的结合构象图
- 生成与批准药物（吉非替尼、厄洛替尼）的性质比较表
- 撰写包含方法论、结果和建议的科学摘要
- 导出带正确引用的 PDF

预期输出：
- 10-20 个新型 EGFR 抑制剂候选物的排序列表
- 预测的活性和 ADMET 性质
- 对接构象和结合分析
- 带发表级图表的综合科学报告
```

### 案例 2：罕见疾病的药物重定位

**目标**：识别可用于治疗罕见代谢紊乱的 FDA 批准药物。

**使用的技能**：
- `database-lookup` — DrugBank、Open Targets、STRING、KEGG、Reactome、ClinicalTrials.gov、FDA
- `paper-lookup` — OpenAlex、bioRxiv、PubMed
- `networkx` — 网络分析
- `bioservices` — 生物数据库查询
- `literature-review` — 系统综述

**工作流程**：

```
步骤 1：定义疾病通路
- 查询 KEGG 和 Reactome 获取疾病相关通路
- 识别关键蛋白质和酶
- 映射上游和下游通路组分

步骤 2：查找蛋白质-蛋白质相互作用
- 查询 STRING 数据库获取相互作用伙伴
- 使用 NetworkX 构建关键疾病蛋白周围的 PPI 网络
- 识别枢纽蛋白和瓶颈
- 计算中心性指标（介数、接近度）

步骤 3：使用 Open Targets 查询可药物靶点
- 搜索与疾病表型相关的靶点
- 按临床先例和可成药性过滤
- 优先选择有现有批准药物的靶点

步骤 4：在 DrugBank 中搜索靶向识别蛋白的药物
- 查询批准药物及其靶点
- 按与疾病相关的作用机制过滤
- 检索药物性质和安全性信息

步骤 5：查询 FDA 数据库获取安全性概况
- 检查 FDA 不良事件数据库（FAERS）
- 审查药品标签和黑框警告
- 评估罕见疾病人群的风险-收益

步骤 6：搜索 ClinicalTrials.gov 获取先前重定位尝试
- 查询疾病名 + 药物名
- 检查失败试验（及失败原因）
- 识别可能竞争的正在进行试验

步骤 7：进行通路富集分析
- 将药物靶点映射到疾病通路
- 使用 Reactome 计算富集分数
- 识别影响多个通路节点的药物

步骤 8：进行系统文献综述
- PubMed 搜索药物名 + 疾病关联
- 包括 bioRxiv 的近期未发表发现
- 记录任何病例报告或超说明书使用

步骤 9：优先排序候选物
- 按通路相关性、安全性概况、现有证据排序
- 考虑口服可用性、血脑屏障穿透等因素
- 评估商业可行性和专利状态

步骤 10：生成重定位报告
- 创建药物-靶点-疾病关系网络可视化
- 生成前 5 个候选物的比较表
- 为每个候选物撰写详细理由
- 包含作用机制图
- 提供临床前验证建议
- 格式化为带引用的专业 PDF

预期输出：
- 5-10 个重定位候选物的排序列表
- 药物-靶点-疾病关系网络分析
- 安全性和有效性证据摘要
- 含下一步行动的重定位策略报告
```

## 癌症基因组学与精准医学

### 案例 3：临床变异解读流程

**目标**：分析患者肿瘤测序数据以识别可操作的突变和治疗建议。

**使用的技能**：
- `database-lookup` — Ensembl、ClinVar、COSMIC、NCBI Gene、UniProt、ClinPGx、DrugBank、ClinicalTrials.gov、Open Targets
- `paper-lookup` — PubMed 文献证据检索
- `pysam` — 解析 VCF 文件
- `gget` — 统一基因/蛋白数据检索
- `clinical-reports` — 生成临床报告 PDF

**工作流程**：

```
步骤 1：解析和过滤 VCF 文件
- 使用 pysam 读取肿瘤 VCF
- 过滤高质量变异（QUAL > 30，DP > 20）
- 提取变异位置、等位基因和 VAF
- 分离 SNV、indel 和结构变异

步骤 2：使用 Ensembl 注释变异
- 查询 Ensembl VEP API 获取功能后果
- 分类变异：错义、无义、移码、剪接位点
- 提取转录本信息和蛋白质变化
- 识别每个基因的规范转录本

步骤 3：查询 ClinVar 获取已知致病变异
- 按基因组坐标搜索 ClinVar
- 提取临床意义分类
- 注意冲突解读和审查状态
- 优先处理"致病性"或"可能致病性"标签的变异

步骤 4：查询 COSMIC 获取体细胞癌症突变
- 为每个变异搜索 COSMIC
- 提取跨癌症类型的突变频率
- 识别热点突变（高复发）
- 记录药物耐药突变

步骤 5：从 NCBI Gene 检索基因信息
- 获取详细基因描述
- 提取相关表型和疾病
- 识别癌基因 vs 抑癌基因分类
- 记录基因功能和生物学通路

步骤 6：使用 UniProt 评估蛋白质水平影响
- 查询蛋白质域信息
- 将变异映射到功能域（激酶域、结合位点）
- 检查变异是否影响活性位点或蛋白质稳定性
- 检索翻译后修饰位点

步骤 7：搜索 DrugBank 获取可靶向改变
- 查询靶向突变基因的药物
- 过滤 FDA 批准和研究药物
- 提取作用机制和适应症
- 优先处理有批准靶向治疗的变异

步骤 8：使用 Open Targets 查询靶点-疾病关联
- 验证治疗假设
- 评估靶点可成药性评分
- 审查每个基因-疾病对的临床先例

步骤 9：搜索 ClinicalTrials.gov 获取匹配试验
- 构建查询：癌症类型 + 基因名 + 变异
- 过滤：招募状态、II/III 期试验
- 提取试验资格标准
- 记录地理位置和联系信息

步骤 10：文献搜索获取临床证据
- PubMed 查询："[基因] AND [变异] AND [癌症类型]"
- 关注：病例报告、临床结果、耐药机制
- 提取相关预后或预测信息

步骤 11：按可操作性分类变异
- I 级：FDA 批准针对此变异的治疗
- II 级：此变异有可用临床试验
- III 级：不同癌症中批准的治疗
- IV 级：生物学证据但无批准治疗

步骤 12：生成临床基因组学报告
- 关键发现执行摘要
- 带证据级别的可操作变异表
- 带支持证据的治疗建议
- 带资格信息的临床试验选项
- 基于突变谱的预后意义
- 参考指南（NCCN、ESMO、AMP/ASCO/CAP）
- 使用 clinical-reports 技能生成专业 PDF

预期输出：
- 带临床意义的注释变异列表
- 分层的可操作突变列表
- 带证据水平的治疗建议
- 匹配的临床试验
- 综合临床基因组学报告（PDF）
```

### 案例 4：基于基因表达的癌症亚型分类

**目标**：使用 RNA-seq 数据对乳腺癌亚型进行分类，并识别亚型特异性治疗脆弱性。

**使用的技能**：
- `database-lookup` — NCBI Gene、Reactome、Open Targets
- `paper-lookup` — PubMed 文献验证
- `pydeseq2` — 差异表达分析
- `scanpy` — 聚类和可视化
- `scikit-learn` — 机器学习分类
- `gget` — 基因数据检索
- `matplotlib` — 可视化
- `seaborn` — 热图
- `plotly` — 交互式可视化
- `scikit-survival` — 生存分析

**工作流程**：

```
步骤 1：加载和预处理 RNA-seq 数据
- 加载计数矩阵（基因 × 样本）
- 过滤低表达基因（平均计数 < 10）
- 使用 DESeq2 大小因子标准化
- 应用方差稳定转换（VST）

步骤 2：使用 PAM50 基因对样本分类
- 查询 NCBI Gene 获取 PAM50 分类器基因列表
- 提取 PAM50 基因的表达值
- 在标记训练数据上训练随机森林分类器
- 预测亚型：Luminal A、Luminal B、HER2+、Basal、Normal-like
- 使用已发表标记物验证（ESR1、PGR、ERBB2、MKI67）

步骤 3：对每个亚型进行差异表达分析
- 使用 PyDESeq2 比较每个亚型 vs 其他所有亚型
- 应用多重检验校正（FDR < 0.05）
- 按 log2 倍数变化过滤（|LFC| > 1.5）
- 识别亚型特异性特征基因

步骤 4：注释差异表达基因
- 查询 NCBI Gene 获取详细注释
- 分类为癌基因、抑癌基因或其他
- 提取生物学过程和分子功能术语

步骤 5：通路富集分析
- 将基因列表提交到 Reactome API
- 识别每个亚型的富集通路（p < 0.01）
- 关注可药物通路（激酶信号、代谢）

步骤 6：使用 Open Targets 识别治疗靶点
- 为每个上调基因查询 Open Targets
- 按可成药性评分 > 5 过滤
- 优先处理有临床先例的靶点

步骤 7：创建综合可视化
- 生成按亚型着色的所有样本 UMAP 投影
- 创建跨亚型的 PAM50 基因热图
- 生成每个亚型比较的火山图
- 生成通路富集点图

步骤 8：文献验证
- PubMed 搜索每个预测治疗靶点
- 总结临床证据和正在进行试验

步骤 9：生成亚型特异性建议
- 列出前 5 个差异表达基因
- 识别富集的生物学通路
- 基于脆弱性推荐治疗策略

步骤 10：创建综合报告
- 带置信分数的分类结果
- 每个亚型的差异表达表
- 通路富集摘要
- 治疗靶点推荐
- 带引用的发表级图表

预期输出：
- 样本分类为分子亚型
- 亚型特异性基因特征
- 通路富集概况
- 每个亚型优先治疗靶点
```

## 单细胞转录组学

### 案例 5：肿瘤微环境的单细胞图谱绘制

**目标**：对肿瘤活检样本进行单细胞 RNA-seq 分析，以表征肿瘤微环境中的细胞类型和状态。

**使用的技能**：
- `scanpy` — 单细胞分析
- `scvi-tools` — 批次校正和整合
- `cellxgene-census` — 参考数据
- `scvelo` — RNA 速度
- `arboreto` — 基因调控网络
- `gget` — 基因查询
- `database-lookup` — NCBI Gene、UniProt、STRING
- `scikit-learn` — 聚类评估
- `matplotlib`、`seaborn`、`plotly` — 可视化

**工作流程**：

```
步骤 1：加载和质控单细胞数据
- 使用 Scanpy 读取 10x Genomics 数据
- 计算 QC 指标：n_genes、n_counts、线粒体百分比
- 过滤低质量细胞（< 200 个基因，> 20% 线粒体）
- 过滤低频基因（< 3 个细胞中表达）

步骤 2：标准化和特征选择
- 总计数标准化到每细胞 10,000
- 对数转换（log1p）
- 识别高变异基因（HVGs，top 2000）

步骤 3：降维和批次校正
- 使用 scVI 进行降维和批次校正（如果有多个批次）
- 或使用 Scanpy 的 PCA + Harmony
- 计算邻居图
- 运行 UMAP 进行可视化

步骤 4：聚类和细胞类型注释
- 使用 Leiden 算法进行聚类（分辨率 0.5-1.0）
- 查询 Cellxgene Census 获取参考细胞类型标记
- 使用经典标记物注释：
  * 肿瘤细胞：EPCAM、KRT19
  * T 细胞：CD3D、CD3E
  * B 细胞：CD79A、MS4A1
  * 巨噬细胞：CD68、CD14
  * 成纤维细胞：COL1A1、PDGFRA
  * 内皮细胞：PECAM1、VWF
- 使用 scvi-tools 的 scANVI 进行半监督注释

步骤 5：使用 RNA 速度推断细胞动态
- 使用 scVelo 计算 RNA 速度
- 推断轨迹方向
- 识别分化路径

步骤 6：使用 Arboreto 推断基因调控网络
- 提取每个细胞类型的表达矩阵
- 运行 GRNBoost2 推断 TF-靶标关系
- 识别细胞类型特异性调控因子

步骤 7：差异表达分析
- 比较肿瘤细胞 vs 正常上皮细胞
- 比较肿瘤相关巨噬细胞 vs 正常巨噬细胞
- 识别标志物基因和通路

步骤 8：查询数据库获取功能注释
- 使用 gget 查询差异基因的通路信息
- 使用 STRING 查询蛋白质相互作用
- 使用 UniProt 获取蛋白质功能

步骤 9：创建综合可视化
- UMAP 按细胞类型着色
- 点图显示每个簇的标记基因
- 热图显示标志物表达
- RNA 速度矢量场
- GRN 网络可视化

步骤 10：生成报告
- 细胞类型组成摘要
- 每个簇的标记基因表
- 差异表达结果
- 通路富集分析
- 动态推断摘要
- 带所有图表的 PDF 报告

预期输出：
- 带注释的细胞类型 UMAP
- 每个细胞群体的标志物基因
- 肿瘤细胞 vs 正常细胞的差异表达
- 肿瘤微环境 GRN
- 细胞状态动态推断
```
