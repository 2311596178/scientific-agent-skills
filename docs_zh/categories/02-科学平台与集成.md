# 分类二：科学平台与集成

> 本分类包含 **12 个技能**，涵盖实验室信息管理系统（LIMS）、基因组学云平台、电子实验笔记本（ELN）、显微镜数据管理、工作流平台、无服务器云计算和 GPU 加速等基础设施集成。

---

## 分类概述

现代科学研究越来越依赖于专业的云平台和实验室自动化系统。本分类的技能使 AI Agent 能够与 Benchling、DNAnexus、LatchBio 等科研平台交互，控制 Opentrons 和 PyLabRobot 等实验室自动化设备，以及利用 Modal 等无服务器云平台进行大规模计算。

---

## 技能清单

### 1. benchling-integration（Benchling 研发平台集成）

**一句话简介**：与 Benchling 研发平台集成，通过 Python SDK 和 REST API 程序化访问实验室数据管理。

**详细描述**：
Benchling 是广泛使用的生命科学研发云平台。该技能提供对 Benchling 核心功能的完整访问：注册实体（DNA 序列、蛋白质）、库存系统（样品、容器、位置）、电子实验笔记本（条目、实验方案）、工作流（任务、自动化）以及数据导出。支持通过 Python SDK 进行编程式操作和 REST API 进行数据交互。

**典型使用场景**：
- 批量导入序列和蛋白质数据到 Benchling
- 自动化实验笔记本条目创建
- 库存样品的位置追踪和查询
- 实验工作流的自动化执行

---

### 2. dnanexus-integration（DNAnexus 云平台集成）

**一句话简介**：在 DNAnexus 云平台上进行基因组学和生物医学数据分析。

**详细描述**：
DNAnexus 是专注于基因组学和生物医学数据的云平台。该技能涵盖构建和部署应用/小应用（Python/Bash）、管理数据对象（文件、记录、数据库）、运行分析和工作流、使用 dxpy Python SDK，以及配置应用元数据和依赖（dxapp.json 设置、系统包、Docker、资产）。支持处理 FASTQ/BAM/VCF 文件、生物信息学流程、作业执行、工作流编排，以及项目管理与权限设置等平台操作。

**典型使用场景**：
- 在云端部署和运行生物信息学流程
- 管理大规模基因组数据集
- 构建可复用的基因组分析应用
- 多团队协作的项目管理和权限控制

### 3. opentrons-integration（Opentrons 实验室自动化集成）

**一句话简介**：创建、编辑和调试 Opentrons Python Protocol API v2 协议，实现 Flex 和 OT-2 机器人的自动化液体处理。

**详细描述**：
Opentrons 是实验室自动化领域的开源领导者。该技能支持 Python Protocol API v2 的完整开发流程，包括自动化液体处理、移液工作流、硬件模块控制（热循环仪、温控模块、磁力模块、振荡加热模块、吸光度读板器）、实验器具管理和复杂的生物/化学实验协议开发。

**典型使用场景**：
- 开发高通量筛选的自动化移液协议
- 自动化样品制备和稀释流程
- PCR 反应体系的自动化配制
- 多步骤生化实验的机器人执行

---

### 4. ginkgo-cloud-lab（Ginkgo 云实验室）

**一句话简介**：在 Ginkgo Bioworks 云实验室提交和管理协议，在可重构自动化推车（RAC）上实现自主实验执行。

**详细描述**：
Ginkgo Cloud Lab (cloud.ginkgo.bio) 是一个基于 Web 的自主实验室执行平台。支持三种标准协议：无细胞蛋白表达验证（39 美元/样品，5-10 天周转）、无细胞蛋白表达优化（199 美元/样品，24 个条件的实验设计，6-11 天）和荧光像素艺术生成（25 美元/板，11 种荧光大肠杆菌菌株的细菌艺术品，5-7 天）。包含 EstiMate AI 智能体，用于评估自定义协议的可行性和定价。

**典型使用场景**：
- 快速验证蛋白质表达条件
- 通过实验设计优化蛋白质表达
- 创建荧光细菌艺术品用于科普或教育
- 探索合成生物学实验的自动化执行

### 5. labarchive-integration（LabArchives 电子实验笔记本集成）

**一句话简介**：通过 REST API 与 LabArchives 电子实验笔记本交互，实现笔记本的备份、检索和管理。

**详细描述**：
LabArchives 是广泛使用的电子实验笔记本（ELN）平台。该技能提供对其 REST API 的完整访问：笔记本（备份、检索、管理）、条目（创建、评论、附件）、用户认证、站点报告和分析，以及第三方集成（Protocols.io、GraphPad Prism、SnapGene、Geneious、Jupyter、REDCap）。包含用于配置设置、笔记本操作和条目管理的 Python 脚本。支持多区域 API 端点（美国、英国、澳大利亚）和 OAuth 认证。

**典型使用场景**：
- 批量备份实验笔记本数据
- 自动化实验条目的创建和更新
- 跨平台数据同步和整合
- 实验室数据的程序化检索和分析

### 6. open-notebook（开放式研究笔记本）

**一句话简介**：自托管、开源的 Google NotebookLM 替代方案，用于 AI 驱动的研究和文档分析。

**详细描述**：
Open Notebook 将研究材料组织成笔记本，支持摄取多种内容源（PDF、视频、音频、网页、Office 文档），生成 AI 驱动的笔记和摘要，创建多人播客，实现文档聊天（上下文感知 AI），以及通过全文和向量搜索跨材料搜索。支持 16+ AI 提供商（OpenAI、Anthropic、Google、Ollama、Groq、Mistral），通过自托管确保完全的数据隐私。

**典型使用场景**：
- 构建个人或团队的研究知识库
- 从大量文献中自动生成播客摘要
- 与 PDF 论文进行对话式交互
- 多模态研究材料的统一管理

---

### 7. latchbio-integration（LatchBio 工作流平台集成）

**一句话简介**：在 Latch 平台上构建、部署和执行生物信息学工作流。

**详细描述**：
Latch 平台支持使用 Python 装饰器创建无服务器生物信息学流程、部署 Nextflow/Snakemake 流程、管理云数据（LatchFile、LatchDir）和结构化注册表（Projects、Tables、Records）、配置计算资源（CPU、GPU、内存、存储），以及使用预构建的 Latch Verified 工作流（RNA-seq、AlphaFold、DESeq2、单细胞分析、CRISPR 编辑）。支持自动容器化、UI 生成、工作流版本控制和可扩展云基础设施上的执行。

**典型使用场景**：
- 快速部署生物信息学分析流程到云端
- 使用预构建工作流进行标准分析（如 RNA-seq）
- 构建可复现和可分享的计算流程
- 无需 DevOps 知识的云原生生物信息学开发

### 8. omero-integration（OMERO 显微镜数据管理系统集成）

**一句话简介**：通过 Python 与 OMERO 显微镜数据管理系统交互，实现显微镜图像的综合访问和分析。

**详细描述**：
OMERO 是显微镜数据管理的行业标准平台。该技能提供对存储在 OMERO 服务器中的显微镜图像的全面访问：数据集和筛选数据检索、像素数据分析、注释和元数据管理、感兴趣区域（ROI）的创建和分析、批量处理、OMERO.scripts 开发，以及 OMERO.tables 结构化数据存储。对高内涵筛选数据、多维显微镜数据集和协作图像库的研究至关重要。

**典型使用场景**：
- 批量下载和分析显微镜图像
- 自动化 ROI 标注和测量
- 高内涵筛选数据的批量处理
- 多用户协作的图像数据管理

### 9. protocolsio-integration（Protocols.io 协议管理集成）

**一句话简介**：通过 API 管理科学协议，支持协议的发现、创建、更新和 DOI 发布。

**详细描述**：
Protocols.io 是科学协议管理和共享平台。该技能支持协议的程序化发现（按关键词、DOI、类别搜索）、协议生命周期管理（创建、更新、DOI 发布）、分步骤程序文档、工作空间和讨论的协作开发、文件管理（上传数据、图像、文档）、实验追踪和文档，以及数据导出。支持 OAuth 认证、协议 PDF 生成、材料管理、线程评论、工作空间权限和机构协议库。

**典型使用场景**：
- 自动化协议的版本控制和 DOI 发布
- 将实验方案从实验室系统同步到 Protocols.io
- 构建机构级协议库
- 协议的标准化和可重复性管理

### 10. modal（无服务器云计算平台）

**一句话简介**：在 Modal 上以最小配置运行 Python 代码，自动扩展的 GPU 加速无服务器计算。

**详细描述**：
Modal 是专为 AI/ML 工作负载和科学计算设计的无服务器云平台。支持在强大的 GPU（T4、L4、A10、A100、L40S、H100、H200、B200、B200+）上执行函数，从零自动扩展到数千个容器，按实际使用量付费。关键特性包括：声明式容器镜像构建（推荐 uv/pip/apt 包管理）、自动扩缩容、GPU 加速（每容器最多 8 个 GPU，最多 1,536 GB 显存）、持久存储卷（v1 和 v2）、密钥管理、定时作业（cron）、无服务器 API 部署（FastAPI、ASGI、WSGI、WebSockets）、并行执行 `.map()`、输入并发和动态批处理。免费套餐每月包含 30 美元积分。

**典型使用场景**：
- 部署和推理大型 ML 模型（LLM、图像生成、嵌入）
- GPU 加速的模型训练和微调
- 大规模数据集的并行批处理
- 定时计算密集型作业
- 蛋白质折叠和计算生物学工作负载

### 11. optimize-for-gpu（GPU 加速优化）

**一句话简介**：使用 NVIDIA RAPIDS 生态系统将 Python 代码 GPU 加速，实现 10x-1000x 性能提升。

**详细描述**：
该技能涵盖 12 个 GPU 库的选择决策框架：CuPy（NumPy/SciPy 的即插即用替代品）、Numba CUDA（自定义 GPU 内核）、Warp（JIT 编译模拟内核）、cuDF（pandas 替代品）、cuML（scikit-learn 替代品）、cuGraph（NetworkX 替代品）、KvikIO（GPUDirect 存储）、cuxfilter（GPU 加速交互式仪表板）、cuCIM（scikit-image 替代品 + 数字病理快速全切片读取）、cuVS（GPU 加速向量/相似性搜索）、cuSpatial（GeoPandas 替代品）和 RAFT/pylibraft（稀疏特征求解器、设备内存管理）。所有库通过 CUDA Array Interface 实现零拷贝数据共享。包含优化工作流（先分析性能、评估 GPU 适用性、从即插即用替代品开始、最小化主机-设备传输）、代码转换模式、内存管理原则和常见陷阱。

**典型使用场景**：
- 将现有 NumPy/pandas/scikit-learn 工作负载迁移到 GPU
- 大规模网络分析和图计算加速
- 高维向量搜索和 RAG 系统加速
- 地理空间分析和轨迹分析 GPU 化
- 数字病理图像的快速处理

---

### 12. get-available-resources（计算资源检测）

**一句话简介**：在任何计算密集型科学任务开始时自动检测可用计算资源并生成策略建议。

**详细描述**：
该技能自动识别 CPU 能力、GPU 可用性（NVIDIA CUDA、AMD ROCm、Apple Silicon Metal）、内存限制和磁盘空间。创建包含资源信息和处理建议的 JSON 文件：并行处理（joblib、multiprocessing）、核外计算（Dask、Zarr）、GPU 加速（PyTorch、JAX）或内存高效策略。

**典型使用场景**：
- 在数据分析、模型训练或大文件操作前确定最佳计算策略
- 自动选择 CPU 并行或 GPU 加速方案
- 在资源受限环境中优化内存使用
- 编写自适应计算资源的工作流
